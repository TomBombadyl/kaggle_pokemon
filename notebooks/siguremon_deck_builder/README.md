# siguremon — Deck Builder Tool

**Kaggle:** [siguremon/deck-builder-tool](https://www.kaggle.com/code/siguremon/deck-builder-tool)  
**Competition input:** `pokemon-tcg-ai-battle` (attached in kernel metadata)  
**Pulled locally:** `deck-builder-tool.ipynb` + extracted static UI (`index.html`, `app.js`, `style.css`)

Browser UI to search the official card pool, build a 60-card list, validate copy limits, and export **`deck.csv`** (one card ID per line — same format as Kaggle submissions).

---

## What it does

| Feature | Detail |
|---------|--------|
| Card browser | Search by name, ID, effect, type; filter by kind / energy type |
| Languages | **EN** (`EN_Card_Data.csv`) or **JP** (`JP_Card_Data.csv`) |
| Deck rules enforced in UI | 60 cards; max 4 per name (except Basic Energy); max 1 ACE SPEC |
| Import | Paste existing `deck.csv` (newline or comma separated IDs) |
| Export | Download `deck.csv` or copy to clipboard when validation = OK |
| Sample deck | Loads competition `sample_submission/deck.csv` |

**Not included:** evolution-line sanity, engine `battle_start` — use `scripts/validate_deck.py` for that before packaging.

---

## On Kaggle (author workflow)

1. Open the notebook and attach competition **pokemon-tcg-ai-battle**.
2. **Run All** — cells write `index.html`, `app.js`, `style.css` and copy CSVs to `/kaggle/working/`.
3. **Save Version → Save & Run All (Commit)**.
4. **Output tab → Download** (`index.html`, `app.js`, `style.css`, `EN_Card_Data.csv`, `JP_Card_Data.csv`, optional `deck.csv`).
5. Serve locally (see below) or open via a static server.

**Note:** upstream `index.html` links `styles.css` but the notebook writes `style.css`. Our local extract fixes that link; if downloading raw Kaggle outputs, rename `style.css` → `styles.css` *or* fix the `<link>` tag.

## Local use (this repo)

### 1. Refresh kernel (optional)

```powershell
kaggle kernels pull siguremon/deck-builder-tool -p notebooks/siguremon_deck_builder -m
python scripts/export_siguremon_deck_builder.py
```

### 2. Card data

Copy competition CSVs into the deck-builder folder (after you have them under `data/`):

```powershell
Copy-Item data/EN_Card_Data.csv notebooks/siguremon_deck_builder/
Copy-Item "data/JP Card Data.csv" notebooks/siguremon_deck_builder/JP_Card_Data.csv -ErrorAction SilentlyContinue
```

Or fetch sim data once: `python scripts/fetch_sim_engine.py` (includes card CSVs when available).

### 3. Serve the UI

```powershell
cd notebooks/siguremon_deck_builder
python -m http.server 8080
```

Open [http://localhost:8080](http://localhost:8080) → build deck → **deck.csv** download.

### 4. Validate before submit

```powershell
# Move exported deck into agent_decks or point --deck at download path
python scripts/validate_deck.py --deck agent_decks/my_new_deck.csv
```

Pass → copy into packaging:

```powershell
python scripts/package_starmie.py --deck agent_decks/my_new_deck.csv
# or package_submission.py for other archetypes
```

## Workflow with repo decks

1. **Load sample** or **paste** an existing list (e.g. `agent_decks/starmie_froslass_ashleysandlin.csv`).
2. Edit counts in the UI (search by card name / ID).
3. Export `deck.csv` → save as `agent_decks/<name>.csv`.
4. `python scripts/validate_deck.py --deck agent_decks/<name>.csv`
5. Gate the archetype (`scripts/gate_starmie.py`, `gate_archaludon.py`, etc.).
